CV Formatting Guidelines
  • Use a clean, professional font: Times New Roman, Garamond, or the LaTeX default (Computer Modern).
  • Font size 10–12pt; margins 0.5–1 inch on all sides.
  • Use a consistent date format throughout — e.g., "Sep 2022 – Jun 2024".
  • List entries in reverse chronological order within every section.
  • Bold your name wherever it appears in publication citations.
  • Keep to 2–3 pages maximum for student / early-researcher CVs.
  • Always save and submit as PDF — never send a .docx file to a professor or program.
  • Start every bullet point with a strong action verb — analyzed, developed, designed, implemented.
  • Follow the formula: Action Verb + Core Content + Result/Impact (e.g., "Developed X, achieving Y").
  • Quantify results wherever possible — percentages, counts, rankings, publication metrics.
  • Use present tense for current positions and past tense for previous ones.

Bullet Point Formula (from Rice CCD Guide)
Action Verb + Core Content + Result / Purpose / Impact
Example: "Developed a transformer-based NER model for Bangla text, achieving 91.3% F1 score and surpassing prior state-of-the-art by 4.2 percentage points."
Action Verb Quick Reference
Research: analyzed, investigated, designed, evaluated, synthesized, formulated, tested, validated
Technical: developed, implemented, programmed, computed, engineered, automated, optimized
Communication: presented, published, authored, lectured, reported, demonstrated, articulated
Leadership: coordinated, supervised, mentored, directed, managed, spearheaded, pioneered
Teaching: instructed, tutored, facilitated, assessed, coached, trained, guided

6
Personal Website Setup Guide
1
Create a GitHub Account
  • Go to github.com and sign up for a free account.
  • Choose a professional username — e.g., firstname-lastname or firstnamelastname.
2
Choose a Template
  • A academicpages.github.io — Recommended. Jekyll-based, widely used in academia.
  • B al-folio — Feature-rich, excellent built-in publications support.
  • C Minimal theme — Simpler and lighter; less setup required.
3
Fork & Rename the Repository
  • Navigate to the template repository on GitHub.
  • Click Fork (top-right corner).
  • Rename the forked repo to exactly yourusername.github.io.
4
Configure Your Site
Edit _config.yml in your repository:
title: Your Name
description: PhD Candidate | Computer Science | BUET
url: https://yourusername.github.io
twitter_username: yourhandle
linkedin_username: yourprofile
5
Customize Your Pages
  • About page: Write a 2–3 paragraph bio covering your background, research focus, and goals.
  • Publications: Add your papers (journal, conference, preprint, or "in prep").
  • CV: Upload your PDF to the /files/ folder and link it from the nav.
  • Research: Describe your current projects and research interests.
6
Deploy via GitHub Pages
  • Go to your repo → Settings → Pages.
  • Under Source, select Deploy from branch → branch: main.
  • Wait 2–3 minutes for the build to complete.
  • Visit https://yourusername.github.io — your site is live!
7
Website Content Checklist
  • Professional photo — headshot, appropriate background, clear lighting
  • About / Bio section — 2–3 paragraphs covering background, research focus, and goals
  • Research interests and current projects — brief descriptions with keywords
  • Education history — degrees, institutions, years
  • Publications list — if any; include preprints or "in preparation" papers
  • Downloadable CV — PDF link, clearly labeled in the navigation
  • Contact information — email address at minimum
  • Google Scholar link — if applicable; create a profile if you have publications
  • Social / academic links — LinkedIn, ResearchGate, ORCID, GitHub
